Shopify軽量化

Shopifyサイト軽量化は「計測して、重い要素を減らす」が基本

Shopify側がすでに最適化している部分と、自分で改善すべき部分を切り分けると、無駄な作業を減らしやすくなります。

ShopifyではCDNや画像最適化などの基盤最適化がすでに入っているため、軽量化の主戦場はテーマ、画像、アプリ、追加スクリプトです。

特に重要なのは、最初に見える画像を遅延読み込みしないこと、画像サイズを明示すること、不要なJavaScriptや外部タグを減らすことです。

改善は勘ではなく、ShopifyのWeb Performance reportsやPageSpeed Insightsで計測し、ホーム・商品・コレクションページを優先して見るのが実務的です。

1. まずは「何が遅いか」を計測する

Shopifyは、旧来の単一スコアよりも、実際の訪問者データに近い Core Web Vitals を重視しています。これは、読み込みの速さを示す LCP、画面のガタつきを示す CLS、操作への反応を示す INP の3指標です。

ShopifyのWeb Performance reportsでは、実ユーザーの体験ベースで変化を追えます。PageSpeed Insightsも併用できますが、実運用では「ホーム」「商品」「コレクション」の主要ページを優先して比較したほうが改善対象を絞りやすいです。

計測とは、現状を数値で確認することです。体感だけで直すと、効いたのか悪化したのか判断しづらくなります。

2. Shopifyが自動でやってくれることと、手を入れるべきことを分ける

Shopify Help Centerでは、CDN、画像最適化、gzip圧縮、CSS/JavaScriptの最小化などが標準で入っていると案内されています。つまり、「CDNを入れれば速くなる」のような一般論をそのまま当てはめても、Shopifyではすでに実施済みなことが多いです。

一方で、速度を落としやすい主因として、テーマ、インストール済みアプリ、手動で追加した第三者コードが挙げられています。軽量化の本丸は、ここを減らすことです。

また、外部ドメインからCSSやJavaScriptや画像を読んでいるなら、可能な範囲で `assets` 配下に置いて Shopify CDN から配信するほうが有利です。接続先ドメインが増えると、読み込みの待ち時間が増えやすくなります。

3. 画像最適化は「全部lazy」ではなく、見える画像を優先する

Shopify公式の performance blog では、最初に見える大きな画像、つまりLCP画像を lazy load しないことが重要だと説明されています。最初に見える画像まで遅延すると、読み込み完了が遅れ、体感も指標も悪化しやすくなります。

逆に、画面の下にある画像には `loading="lazy"` を使うのが有効です。Shopifyの Theme Check でも、最初から見えない画像にはネイティブ lazy loading を使うことが推奨されています。

さらに `width` と `height` を入れて画像の表示領域を先に確保すると、CLSの改善につながります。`web.dev` でも、画像サイズ未指定はレイアウトずれの原因になりやすいと整理されています。

4. JavaScriptとアプリは「必要な場所だけ読む」に寄せる

Shopify theme の performance best practices では、描画を止める parser-blocking scripts を避け、`defer` や `async` を使うよう案内されています。これは、HTMLの表示を待たせないための基本です。

アプリや追加機能も、全ページで一律に読み込むより、必要なページだけで読み込む設計のほうが有利です。Shopifyの app embed blocks は、必要なページでだけスクリプトを出しやすくする仕組みです。

実務上は、タグマネージャーの不要タグ、使っていないアプリ、重いアニメーション、過剰なセクション数を先に疑うのが近道です。機能が多いほど売上に効くとは限らず、表示遅延で離脱が増えることもあります。

5. 改善の進め方は「主要3ページを、変更前後で比べる」が安全

Shopifyは、テーマやアプリの影響を見るときに、ホーム17%、商品40%、コレクション43%という重みで主要ページを見ています。これは「売上に効きやすいページから見る」という意味でも理にかなっています。

そのため、改善は一気に大量変更するより、アプリ削減、画像修正、スクリプト変更などを小さく分けて、前後比較しながら進めたほうが安全です。Web Performance reports では、アプリ導入やテーマ更新の前後変化も追いやすくなっています。

軽量化の結論は単純で、Shopifyの標準最適化を前提にしつつ、「テーマ」「画像」「アプリ」「第三者コード」を順番に削ることです。新機能を足す前に、今ある重さを減らすほうが再現性の高い改善になりやすいです。
